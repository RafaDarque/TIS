ARP
...
  • ARP -> Address Resolution Protocol
  • Protocolo da camada 4 do modelo TCP/IP
  • MAC -> Media Access Control - Numero de série da NIC
  • Dispositivos enviam pacotes ARP REQUEST por broadcast
  • Dispositivo que conhece a info responde em forma unicast, através do pacote ARP REPLAY
  • Tipos de ataque de ARP: ARP Spoofing e ARP Poisoning
  • API -> Application Programming Interface
  • Tabela ARP - estrutura que mantém os mais recentes mapeamentos de endereços IP em endereços físicos
  • O que é o ARP? É um protocolo
  • OSI -> open system interconnection
  • ARPs passam só até ao router
  • IP broadcast -> acaba em 255
IPCONFIG
...
  • IP -> Internet Protocol
  • DNS -> Domain name system
  • ipconfig /release libera o endereço IP
  • ipconfig /renew renova o endereço IP
  • ipconfig /all exibe todas as informações de configuração TCP/IP
  • ipconfig /flushdns limpar cache dns
  • DHCP - Dynamic Host Configuration Protocol
ROUTE
...
  • route -f - limpa todas as entradas que não estão associadas a interfaces de rede
  • route -4 - força a utilização do IPv4
  • route 10.* - tenta encontrar aqueles que são compatíveis com 10
  • route PRINT - imprime uma rota
  • route ADD; route DELETE; route CHANGE
  • TCP/IP - Transmission Control Protocol / Internet Protocol
TELNET
...
  • teletype network
  • telnet /a - login automatico com nome de user conectado
  • telnet /l username - especifica o nome de user para fazer login
  • telnet /e caracter - especificar o caracter para sair
  • telnet /f filename - especifica o nome do arquivo
  • telnet /t ansi - ansi é um tipo de terminal. podem ser outros
  • telnet [host [port]] - especifica o nome do host ou IP do computador remoto - por norma a porta TCP a ser usada é a 23
  • telnet freecheess.org - aceder a um site de xadrez online
  • "Interagir" com um servidor
NSLOOKUP
...
  • Name server look up
  • usado em sistemas windows e Unix-like (linux, macOS)
  • consulta informações de resolução de DNS
  • nslookup www.google.com - retorna o endereço IP associado ao nome
  • nslookup 8.8.8.8 - retorna o nome associado ao endereço IP
  • nslookup - modo interativo
  • nslookup www.google.com 8.8.8.8 - define um servidor DNS específico
  • nslookup -type=AAAA www.google.com - consulta para um endereço IPv6
  • nslookup -debug www.google.com - exibe informações adicionais - por exemplo header e TTL
  • nslookup -input=arquivodeentrada - definir um arquivo de entrada personalizado - não funciona com windows
PING
...
  • ping -t - envia mensagens até ser interrompido
  • ping -a - exibe o nome do host correspondente
  • ping -n - mensagens eco a serem enviadas - padrao é 4
  • ping -l - especifica tamanho em bytes - o padrao é 32 - o tamanho máximo é 65.527
  • ping -f- mensagens enviadas com sinalizador
  • ping -s 2000 servidor - obter info sobre este servidor e quantos bytes a serem enviados
  • ping -t google.com - obter informações da latência e conectividade do google
  • ping -n 4 google.com - especifica o numero de pacotes
  • ping -l 1000 google.com - envia pacotes de ping com tamanho de 1000bytes
TRACERT
...
  • tracert /h 6 - numero de saltos até ao destino
  • tracert /w 2000 - tempo em ms para aguardar resosta (aparece * se não for recebida)
  • tracert /4 - apenas IPv4
  • tracert /d - nao resolve os nomes
  • É para ver os intermediários no caminho dos pacotes
SSH
...
  • Secure Shell
  • Comunicação entre servidor e cliente
  • Cuida da autenticidade, criptografia e integridade dos dados
  • ssh -i - especifica o ficheiro chave
  • ssh -l :endereco_remoto:porta_remota- cria um tunel SSH local para encminhamento de portas
  • ssh -p - especifica a porta SSH - a porta padrao é 22
  • ssh -p 5785 nomeutiliador@enderecoIPouNomeHost - alterar a porta
  • @ -> at
  • Vem instalado com o sistema operativo
  • exit -> para sair do programa
  • OpenSSH Server - para tornar um computador num servidor
  • Tem interface gráfica
NETSTAT
...
  • netstat -e - exibe estatísticas de ethernet
  • netstat -s - exibe estatísticas por protocolo - padrao mostra TCP, UDP, ICMP, IP
  • netstat -n - exibe as conexoes TCP ativas
  • netstat -s -p tcp udp - exibe estatisticas para os protocolos TCP e UDP
  • netstat -b - exibe conexoes TCP ativas e qual o programa a utilizar
  • netstat -a - exibe todas as conexões e portas abertas
  • proto - é o protocolo
FTP
...
  • File transfer protocol
  • transfere arquivos entre dispositivo local e servidor FTP
  • hostname
  • dir ou ls mostrar o que há no local
  • open hostname - conectar ao servidor
  • user username password - para autenticacao
  • cd localizacao - para mudar diretorio
  • lcd localizao - para mudar para outra localização
  • get nomedoarquivoremoto - transfere do servidor para o pc
  • put nomedoarquivoremoto - envia do pc para o servidor
  • quit ou bye - sair do cliente FTP
  • ex: get favicon.ico
  • Porta 21
NBTSTAT
...
  • NetBIOS Statistics
  • relacionado com protocolo NetBIOS
  • lista conexões NetBIOS
  • Ex: nbtstat /a corp07
  • Ex: nbtstat /A 10.0.0.99
  • nbtstat /s - exibe sessoes de cliente e servidor NetBIOS, com nomes
  • nbtstat /S - exibe as sessoes apenas com os IP

  • WLAN - ligacao de todos os dispositivos wireless do computador
  • É possivel executar um documento com extensão .bat
  • RPC - remote procedure call

-> O comando pathping combina a funcionalidade de dois outros comandos. Quais são?
ping e tracert

-> Por vezes o comando pathping não lista os nodes (nós) para além do router. Uma das razões para isto acontecer é o facto de alguns routers terem um protocolo bloqueado. Qual é este protocolo?
ICMP

->3: Escreva um comando pathping para o DNS google, com as seguintes regras:
- número máximo de saltos: 10
- não resolver endereços IP
- número máximo de pedidos: 4
- espera em milissegundos para cada resposta: 400
- forçar o uso do IPv4
Escreva pela ordem das regras.

pathping -h 10 -n -q 4 -w 400 -4 8.8.8.8
